# CourtFlow 🏀

An NBA player catalog Android app built with Kotlin, Jetpack Compose and a multi-module Clean Architecture. Browse NBA players in an endlessly scrolling list, open a player's detail and jump straight to the detail of the team they play for.

Player data is provided by the [balldontlie](https://app.balldontlie.io/) API.

## Features

- **Player list** — paginated list of NBA players (35 records per page, loaded automatically as you scroll) showing name, position and current team
- **Offline-first caching** — every fetched screen (player list, team list, player & team detail, recent games) is cached in Room and served offline; data is refetched only when its cache is older than **1 day** (a uniform TTL) or on pull-to-refresh. The unfiltered player list pages from Room through a Paging 3 `RemoteMediator`; when the network is down an offline banner appears over the cached data instead of an error screen. Player search and team rosters stay network-only by design
- **Player search** — debounced search field backed by the API's `search` query parameter, with a clear action and a dedicated empty state when no players match
- **Player detail** — full player information with a link to their team
- **Team list** — all 30 NBA teams grouped by conference and division with sticky headers, reachable from the player list top bar
- **Team detail** — team information (city, abbreviation, conference, division) served from the offline-first Room cache (refreshed when older than a day), plus the team roster and the team's recent games (last 45 days, via the `/games` endpoint, also cached per team)
- **Pull-to-refresh** — on the player list, the team list and the team roster
- **Skeleton loading** — the player list shows skeleton placeholders instead of a first-load spinner
- **Deep links & shortcuts** — `courtflow://player/{id}`, `courtflow://team/{id}`, `courtflow://players` and `courtflow://teams` deep links, plus static launcher shortcuts
- **Share** — player and team details can be shared via the system share sheet
- **Error handling** — failed first loads show a message with a Retry button instead of a blank screen

## Images

The balldontlie API returns **no images** — no player photos and no team logos, in any pricing tier (verified against both the docs and the live API). The app therefore generates deterministic placeholder art via [DiceBear](https://www.dicebear.com) (free, keyless): the player/team **id is used as the generation seed**, so the same entity always renders the same avatar/emblem.

Images are loaded with **Glide Compose** with a loading placeholder icon, an error-state icon and a circular crop. Screenshot tests pass an empty `imageUrl` so goldens stay deterministic.

## Architecture

```
app ─────► feature:players ──► domain ──► core:common
     ────► feature:teams   ──► core:designsystem
     ────► data ──► domain, core:network
```

- **`domain`** — business models, repository interfaces and use cases (pure Kotlin logic)
- **`data`** — repository implementations, DTO→domain mappers, Paging 3 sources and the Room cache (entities, DAOs, `RemoteMediator`); reads are offline-first with a uniform 1-day TTL tracked in a `cache_metadata` table
- **`core:network`** — networking stack; the Retrofit interface and Moshi DTOs are generated at build time by OpenAPI Generator from the official balldontlie definition (`core/network/openapi/nba.yml`)
- **`core:designsystem`** — shared Material 3 theme and reusable components
- **`feature:*`** — one module per feature; each screen follows the MVI contract pattern (`State` / `Intent` / `Effect`)

## Getting started

### Prerequisites

- Android Studio (latest stable)
- JDK 11+
- A free API key from [balldontlie.io](https://app.balldontlie.io/)

### Setup

1. Clone the repository and open it in Android Studio.
2. Add your API key to `local.properties` in the project root:

   ```properties
   balldontlie.apiKey=YOUR_API_KEY
   ```

   The key is injected into `BuildConfig` of the `:core:network` module at build time.

> [!IMPORTANT]
> Without a valid API key the app builds fine but every request returns `401 Unauthorized`, so all lists stay empty.

3. Build and run:

   ```bash
   ./gradlew assembleDebug
   ```

## Development

```bash
./gradlew test                                       # run all unit tests
./gradlew :domain:testDebugUnitTest                  # run tests of a single module
./gradlew detekt ktlintCheck                         # static analysis
./gradlew :core:designsystem:verifyRoborazziDebug    # screenshot tests against goldens
./gradlew :core:designsystem:recordRoborazziDebug    # re-record goldens after intended visual changes
```

Architecture rules (package placement of ViewModels and UseCases) are enforced by Konsist tests:

```bash
./gradlew :app:testDebugUnitTest --tests "cz.vanama.courtflow.konsist.ArchitectureTest"
```

### Generated API client

The Retrofit interface (`NBAApi`) and Moshi models are **generated, not hand-written**.
The source of truth is the official balldontlie OpenAPI definition, a snapshot of
[`balldontlie.io/openapi/nba.yml`](https://www.balldontlie.io/openapi/nba.yml) committed at
`core/network/openapi/nba.yml`. Generation runs automatically as part of every build
(`openApiGenerate` task, output in `core/network/build/generated/openapi` — never edit it
or commit it).

```bash
./gradlew :core:network:openApiGenerate    # (re)generate the client explicitly
./gradlew :core:network:updateApiSpec      # refresh the committed spec snapshot from balldontlie.io
```

After `updateApiSpec`, review the spec diff and rebuild — compile errors highlight any
breaking upstream changes. The committed snapshot keeps builds deterministic and offline-friendly.
